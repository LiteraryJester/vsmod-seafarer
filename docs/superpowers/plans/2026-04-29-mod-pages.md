# Mod Pages Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce two self-contained HTML mod-hub pages — one for Seafarer (full rewrite of v0.2.4 page) and one for the new Progression Framework (brand-new) — pasteable into the Vintage Story mod hub editor.

**Architecture:** Each page is a single `<div>`-wrapped HTML document with inline styles, mirroring the existing Salt-and-Sand page outer style. Top-level sections use emoji-prefixed `<h3>` headings (BetterRuins/PrimitiveSurvival style). Server-config kept in a collapsible `<div class="spoiler">`. Both pages reference the same visual conventions documented in the design spec.

**Tech Stack:** HTML only. No build pipeline, no JS. Browser preview for QA. Validators (`validate-assets.py` etc.) do not apply — these files live outside `assets/`.

**Spec:** `docs/superpowers/specs/2026-04-29-mod-pages-design.md`

**Reference data already collected:**
- Seafarer current version: **0.5.0** (1.22.0-rc.7+); previous 1.21 backport: **0.1.4**
- Progression Framework current version: **1.0.0** (1.22.0-rc.7+)
- Seafarer hard-deps: `progressionframework 1.0.0`
- Optional compat: ConfigLib, Hydrate or Diedrate, Expanded Foods (carry over from old page)
- Tortuga NPCs (canonical from `quests/Tortuga.md` + `quests/quest-tree.md`): Morgan (provisioner, optimistic), Drake (shipwright, gruff but practical), Dawn Marie (plantation owner, depressed but hopeful), Celeste (retired pirate, cynical/world-weary), Potato King (mad miner)
- Rebuilding scoreboard: 0 (Destitute) → 4 (Struggling); 4 main quests each award +1
- Five professions: Carpentry/Shipwright, Hunting/Bear Hunter, Combat/Buccaneer, Gardening/Gardener, Cooking/Brewer + Pie Master (from `assets/seafarer/config/training/professions.json`)
- Trait-gating real example: `seafarer:assets/seafarer/recipes/grid/outrigger-rollers.json` — two variants, second has `requiresTrait: "shipwright"`

## Visual Conventions (apply to BOTH pages)

Outer wrapper:

```html
<div style="max-width: 820px; margin: 0 auto; font-family: sans-serif; line-height: 1.6;">
  ... content ...
</div>
```

Section divider:

```html
<hr style="border: none; border-top: 1px solid #555; margin: 1.5em 0;" />
```

Top-level section heading:

```html
<h3>⚓ Seafaring</h3>
```

Sub-section heading: plain `<h4>`.

Feature bullets (canonical pattern from old page):

```html
<li><strong>Feature Name</strong> &mdash; One-sentence description.</li>
```

Spoiler block (config table):

```html
<div class="spoiler">
  <div class="spoiler-toggle">Server Configuration</div>
  <div class="spoiler-text">
    ... table goes here ...
  </div>
</div>
```

Code block for JSON examples (PF page only):

```html
<pre style="background: #1e1e1e; color: #d4d4d4; padding: 0.75em 1em; overflow-x: auto; border-radius: 4px;"><code>... JSON ...</code></pre>
```

   - `<h2>` — `<strong>Quests, training, and evolving traders for Vintage Story modders.</strong>`
   - Pitch paragraph: "**Progression Framework** is a content-empty mod that provides three reusable systems for other mods to build on: a quest engine, a training/profession engine, and an evolving-trader engine. Drop JSON into your mod's asset folders and the framework discovers it at load time."
   - Version banner: `Current release : <strong>v1.0.0</strong>` (Vintage Story 1.22.0-rc.7+).

2. **`<h3>✨ For Players</h3>`:**
   - One paragraph: "This mod adds nothing on its own &mdash; install it because a content mod requires it. Currently shipped by: <a href=\"https://mods.vintagestory.at/seafarer\">Seafarer</a>."
   - `<h4>What it gives you in-game</h4>` with three bullets:
     - `<strong>L &mdash; Training Ledger</strong> &mdash; A tabbed dialog showing your training progress and active quests.`
     - `<strong>Quest log</strong> &mdash; Quests track automatically as you talk to NPCs and turn in items.`
     - `<strong>Evolving traders</strong> &mdash; NPC stock and dialogue change as you complete work for them.`

3. **`<h3>🧩 For Modders &mdash; Overview</h3>`:**
   - One paragraph: "The framework scans every loaded mod domain for content at <code>AssetsFinalize</code>. Drop JSON into the conventional paths below and you have working content."
   - Table with three columns (Subsystem | Path | Notes):
     - `Quests | <code>&lt;yourmod&gt;:config/quests/*.json</code> | First-loaded wins on collisions.`
     - `Training | <code>&lt;yourmod&gt;:config/training/*.json</code> | Recognised filenames: <code>professions.json</code>, <code>xp.json</code>, <code>config.json</code>.`
     - `Tradelists | <code>&lt;yourmod&gt;:config/tradelists/*.json</code> | Looked up by stem from <code>EntityEvolvingTrader</code> JSONs.`
     - `Dialogue | base-game NPC conversation system | Quests subscribe to dialogue triggers.`
   - `<h4>Registered identifiers</h4>` paragraph: "These names are stable for cross-mod use: entity class <code>EntityEvolvingTrader</code>, item class <code>ItemTrainingBook</code>, hotkey <code>progressionframeworkledger</code>, network channel <code>progressionframework:questsync</code>, Harmony id <code>progressionframework</code>."

4. **`<h3>📋 Adding a Quest</h3>`:**
   - One paragraph: "Save a JSON file under your mod's <code>config/quests/</code>. The <code>code</code> is its identifier; <code>npc</code> is the entity code that gives it. <code>delivery</code> objectives count items the player turns in to that NPC; rewards fire when the threshold is hit."
   - `<pre><code>` block with this exact JSON (12 lines counting braces):

```json
{
  "code": "harvest-help",
  "npc": "mymod:trader-farmer",
  "scope": "server",
  "autoEnable": true,
  "objectives": [{
    "code": "deliver-wheat",
    "type": "delivery",
    "items": [{ "item": "game:grain-spelt", "quantity": 1 }],
    "required": 32,
    "rewards": [{ "type": "addSellingItem", "itemCode": "game:scythe-copper", "itemType": "item", "stocksize": 1, "stockAvg": 1, "priceAvg": 8 }]
  }]
}
```

   - One closing sentence: "Quest titles, descriptions, and group names go through your mod's <code>lang/en.json</code> using the <code>titleLangKey</code>, <code>descriptionLangKey</code>, and <code>groupLangKey</code> fields."

5. **`<h3>🎓 Adding Training</h3>`:**
   - One paragraph: "A <strong>profession</strong> contains one or more <strong>trainings</strong>; each training has <strong>levels</strong>; each level grants a <strong>trait</strong>. Save the list at <code>config/training/professions.json</code>."
   - `<pre><code>` block (~12 lines):

```json
[
  {
    "code": "gardening",
    "color": "#4F7F3C",
    "trainings": [{
      "code": "gardener",
      "levels": [{ "level": 1, "xp": 100, "title": "Gardener", "trait": "green-thumb" }]
    }]
  }
]
```

   - One paragraph: "XP sources go in <code>config/training/xp.json</code>. The <code>craft</code> kind grants XP whenever a matching grid recipe completes; <code>book</code> when an <code>ItemTrainingBook</code> is read; <code>dialogue</code> when a registered dialogue trigger fires."
   - `<pre><code>` block (~6 lines):

```json
[
  { "type": "craft", "training": "gardener", "recipe": "mymod:flowerpot-*", "xp": 5 },
  { "type": "book",  "training": "gardener", "item": "mymod:trainingbook-gardener", "xp": 100 }
]
```

6. **`<h3>🔒 Gating a Recipe with a Crafting Trait</h3>`:**
   - One paragraph: "Once a training level grants a trait (<code>green-thumb</code> in the example above), any grid recipe variant with <code>requiresTrait: \"green-thumb\"</code> only matches when the player has earned that trait. A common pattern is to ship <strong>two variants</strong> &mdash; one ungated with extra ingredients (so untrained players can still build it the hard way), one trait-gated with a simpler ingredient list."
   - `<pre><code>` block (~14 lines, modeled on `outrigger-rollers.json`):

```json
[
  {
    "ingredientPattern": "S,F,R",
    "ingredients": {
      "S": { "type": "item", "code": "mymod:schematic-greenhouse" },
      "F": { "type": "item", "code": "game:firewood",   "quantity": 5 },
      "R": { "type": "item", "code": "game:rope",       "quantity": 4 }
    },
    "width": 1, "height": 3, "shapeless": true,
    "output": { "type": "block", "code": "mymod:greenhouse-frame", "quantity": 1 }
  },
  {
    "requiresTrait": "green-thumb",
    "ingredientPattern": "F,R",
    "ingredients": {
      "F": { "type": "item", "code": "game:firewood", "quantity": 5 },
      "R": { "type": "item", "code": "game:rope",     "quantity": 4 }
    },
    "width": 1, "height": 2, "shapeless": true,
    "output": { "type": "block", "code": "mymod:greenhouse-frame", "quantity": 1 }
  }
]
```

   - One closing sentence: "<code>requiresTrait</code> is a base-game <code>GridRecipe</code> field, not a Progression Framework one &mdash; the framework's contribution is granting the trait via training so the recipe gate fires."

7. **`<h3>🛠 Built-in Handlers</h3>`:**
   - Three short bulleted lists:
